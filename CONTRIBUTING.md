# Contributing Guidelines


Thank you for your interest in contributing! Follow these rules to maintain consistency:


## 1. Folder Structure
Each topic should have its own folder inside `solutions/`.
Each problem must contain:
- `<name>.py` (solution)
- `<name>_test.py` (pytest file)
- `<name>.md` (short explanation)


## 2. Code Style
- Follow PEP8
- Include docstrings
- Use clear variable names


## 3. Tests
All contributions must include pytest cases.


## 4. Pull Requests
- Describe the problem you solved
- Ensure all tests pass (`pytest -q`)
